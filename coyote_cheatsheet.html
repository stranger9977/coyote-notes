COYOTE — concepts cheat sheet

Quick refreshers so I can talk about each intelligently. One or two bullets each.

Q: The 5 outputs of Model A are continuous — what’s the loss function?

  • MSE — mean squared error (nn.MSELoss), with the Adam optimizer. Both models use it: Model A averages MSE across its 5 continuous outputs (Δx, Δy, accel, x-vel, y-vel); Model B is MSE on expected remaining yards.
  • Why MSE: the targets are unbounded continuous numbers; MSE is the natural regression loss — it’s the Gaussian-noise likelihood, so it trains the net to predict the conditional mean. The output layer is linear (no squashing) precisely so it can emit any real number, including negatives.
  • The catch to mention: “predicts the mean” means MSE underweights the heavy tail (the big returns). That’s exactly why the upgrade for Model B is a quantile / pinball loss → a whole distribution instead of a point.

ReLU (Rectified Linear Unit)

  • ReLU(x) = max(0, x) — passes positives, clamps negatives to 0. It’s the nonlinearity placed between layers. Analogy: a one-way valve.
  • Why it’s needed: without a nonlinearity, stacked linear layers collapse into a single linear map — depth would be pointless. ReLU is cheap and doesn’t vanish-gradient for positive inputs.

Linear (identity) activation

  • No activation — the output is just Wx + b passed straight through. Used on the output layer for regression because predictions must be able to be any real number (a ReLU output could never be negative — but velocities and Δx can be).
  • Pattern to remember: hidden layers = ReLU, output layer = linear.

Feed-forward (FFNN)

  • Data flows one direction only: input → hidden → output, no loops back. You can read it straight down the forward() method.
  • Contrast: RNN/LSTM loop (state feeds back, for sequences); a transformer doesn’t loop but lets tokens attend sideways. Note: “feed-forward” describes the forward pass — training still back-propagates the error backward.

MLP (multilayer perceptron)

  • The specific feed-forward net here: fully-connected (nn.Linear) layers + ReLU — every neuron connects to every neuron in the next layer.
  • Both COYOTE nets are MLPs: Model A 42→50→25→10→5, Model B 154→200→100→20→1. The paper’s “FFNN” = the code’s “MLP” — same thing. (The triangle/image CNN in the repo is an abandoned path.)

Transformer / self-attention — and why it ignores player order

  • Treats the 22 players as a set of tokens. Each player forms a query, key, value (via shared WQ/WK/WV); attention weight = softmax(query·key) decides who attends to whom, recomputed every frame. It models pairwise interactions explicitly.
  • Why order-agnostic: the same weights hit every player (weight-sharing) and pooling is symmetric (mean) → permutation-invariant. Reshuffle the players → identical output. No sort needed; it learns “most relevant” instead of hard-coding “nearest.”

Weight-sharing & why the transformer beats the FFNN (the nuance)

  • The FFNN can model interactions (it’s a universal approximator) — the transformer’s edge is inductive bias / sample-efficiency, not raw capability. The FFNN must re-learn the same interaction at every slot, learn the permutation symmetry from data, and untangle a brittle distance-sort; the transformer gets all that for free by reusing one small set of weights across all 22 players (so each player is ~22× the training signal). Analogy: transformer : a set of players :: CNN : an image.
  • The numbers: original MLP 0.639 → gradient-boosted trees 0.674 → transformer 0.702. So roughly half the lift is “better model class,” half is “the right structural prior for a set of interacting agents.”

Bar-fight analogy (the keeper)

  • FFNN bouncer: numbers everyone into fixed clipboard slots by distance to the bar. Re-learns the same headlock once per slot, panics when two people swap order, and his chart thrashes every time anyone moves past anyone — brainpower wasted on bookkeeping.
  • Attention bouncer: takes in the whole room at once; each person focuses on whoever actually threatens/helps them right now; recognizes a headlock anywhere; reorder people → same read. Spends all his attention on who’s engaging whom, so he calls the brawl right with far less experience.
  • LLM tie-in: the same machine, with words as the “people” and attention deciding which earlier words each word looks at — weight-sharing across positions is what lets it scale to 100k tokens. A sentence is just a tiny “field.”

Jensen’s inequality — why a single “average” ghost is biased

  • The rule: for a curved (nonlinear) function V, the value of the average input ≠ the average of the values: V(mean) ≠ E[V]. (Convex V → V(mean) ≤ E[V]; concave → the reverse; equal only if V is a straight line.)
  • Plain version: averaging first then scoring ≠ scoring lots of possibilities then averaging — whenever the scorer bends.
  • Why it bites COYOTE: Model B (expected yards) is nonlinear. The point-ghost plugs in one average position → f(mean ghost), but the honest counterfactual is the average over where the player could be → E[f(ghost)]. They differ, so the point estimate is systematically biased. Ghosting a distribution and averaging Model B over it computes E[f] directly — the Yurko/Ferraz fix.
  • Analogy: a guy who’s “on average at midfield” isn’t worth the value of midfield if he’s sometimes springing a huge block and sometimes whiffing — the average of the outcomes ≠ the outcome of the average.

“Integrate over the distribution” — what that word means

  • It just means take a probability-weighted average (an expected value). “Integrate f over the ghost distribution” = consider every possible ghost position, weight it by how likely it is, and sum the values: E[f] = ∫ f(s)·p(s) ds. The ∫ is simply a continuous sum; the takeaway is “average the score across all plausible ghost spots, weighted by probability.”
  • In practice you never do the calculus — you sample (Monte Carlo): draw, say, 200 ghost positions from Model A’s distribution, run Model B on each, take the mean. That sample-average ≈ the integral E[f(ghost)]; more samples = tighter estimate, and the spread of those samples is your confidence interval.
  • One line: point ghost = f(one position); integrated = average of f(many sampled positions) — the second is the unbiased one (that’s the Jensen fix above).

“Strokes gained” framing for COYOTE itself

  • COYOTE is strokes-gained for special teams: compare what actually happened to what an average player would do from the same spot, credit the difference, sum it over the play. It turns an unobservable (“how good was that block?”) into a difference of two model predictions — no block-grading labels needed.

Pre-catch vs post-catch (don’t confuse with train/inference)

  • It’s a per-model training-data thing, not train-vs-inference. Model A trains on pre-catch frames; Model B trains on post-catch frames (its target, “remaining return yards,” only exists during the return). At inference both run post-catch — COYOTE is only computed on the return.
  • Wrinkle to flag: Model A is trained pre-catch but applied post-catch — a mild train/serve mismatch.

Holdout vs the metric & split-half reliability

  • Model accuracy (corr/MAE, the 0.64→0.70 numbers) is measured on a holdout test set. But COYOTE/ret is computed on all the data, not a holdout — there’s no ground-truth label for “true blocking value,” so you can’t holdout-test the ratings.
  • Instead you check reliability: split each player’s returns in half, correlate the two halves → does the metric reproduce itself? Original MLP ≈ 0.1 (mostly noise) vs transformer ≈ 0.4 (moderate). Strong line: “they validated the model but never validated the metric — and when you do, the original ratings are quite noisy.” Could tighten further with out-of-fold (cross-fit) COYOTE.

How Model A, Model B & the COYOTE calculation work

The whole machine in bullets: Model A builds the “ghost,” Model B scores the field, COYOTE is the gap.

Model A the movement / “ghost” model

Predicts what an average player would do next

MLP   42 → 50 → 25 → 10 → 5  ·  ReLU hidden, linear output  ·  MSE + Adam
  • Job: given the field now, predict where a player will be in the next 0.1 s (Δt) — but trained to capture the average player, so it answers “what would a typical NFL player do here?”
  • Inputs (42): for the target player — x-velocity, y-velocity, acceleration, distance travelled last frame (4); for each of the 5 nearest players — those same features + relative x/y to the target + a same-team flag (7 each = 35); plus the ball’s relative x/y and a return-team indicator (3). The 5 neighbours are listed sorted nearest→furthest.
  • Outputs (5): the target player’s next-frame Δx, Δy, acceleration, x-velocity, y-velocity — all continuous, hence the linear output layer.
  • Why it’s “average”: it’s trained over every player on every return in a season, so it learns generic behaviour, not any one player’s style. That’s deliberate — it’s the yardstick the real player is measured against.
  • How it’s used: run it on the players to roll the field forward 0.1 s → the expected (“ghost”) field state. (Trained on pre-catch frames; applied during the return.)

Model B the value model

Scores how good a field state is for the return

MLP   154 → 200 → 100 → 20 → 1  ·  ReLU hidden, linear output  ·  MSE + Adam
  • Job: given a full field state, predict the expected remaining return yards — how far the returner will still gain from this moment (assuming no penalty/turnover). Call it f(S) for field state S.
  • Inputs (154): the whole field — the ball carrier’s features plus all 21 other players, each as position / velocity / acceleration relative to the ball + a team indicator, again sorted by distance to the ball. (2 ball-to-carrier + 5 carrier + 21×7 others = 154.)
  • Output (1): a single number — expected remaining return yards.
  • Trained on: post-catch frames (the return itself); it also trains on the ghost fields as extra examples, so it’s comfortable scoring counterfactual states too.

The COYOTE calculation per frame → per return → per season

Actual field minus average-player field, summed up

  • One frame, one player: take the actual field S. Build a counterfactual field Ŝ = the same field, but the target player (and anyone in his 3-yd halo) is moved to where Model A says an average player would be. Then:
COYOTEk,t = f(Sk,t) − f(Ŝk,t)   — the expected return yards his real positioning added vs. an average player, on that 0.1 s frame.
  • Sign fix: multiply by −1 for kicking-team players, so positive = above-average for everyone (offense and coverage).
  • Off-ball filter: COYOTE is set to 0 when the player is within 3 yds of the ball carrier (so it never credits the carry/tackle itself — it stays a blocking/coverage metric), and zeroed for extreme-outlier swaps.
  • One return: sum the player’s per-frame COYOTE across every 0.1 s frame of that return → his total COYOTE for the return.
  • One season: average those per-return totals over all returns he played in (≥10 returns to qualify) → COYOTE/ret, the leaderboard stat.
COYOTE/ret = (1/K) · Σk=1..K Σt∈Tk COYOTEk,t   — K = returns the player was in; Tk = the frames of return k.
  • Read it as: “if this player were swapped for an average NFL player in his spot, his team would gain X fewer (or more) return yards per return, on average.”
  • The one-liner: it’s strokes-gained for special teams — actual minus average, accumulated over the play.

How they fit together (say this out loud)

  • Model A makes the ghost (the average-player field) → Model B prices the field (expected return yards) → COYOTE = actual price − ghost price, summed per return and averaged per season.
  • No block-grading labels are ever needed — the credit for an unobservable (“how good was that block?”) falls out as a difference of two model predictions.
